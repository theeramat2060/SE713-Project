# Seed Data Directory

This directory contains structured seed data for populating the database with test records.

## Files

### Data Files
- **adminSeed.ts** - Admin user accounts
- **constituencySeed.ts** - Electoral constituencies (6 total)
- **partySeed.ts** - Political parties (5 total)
- **candidateSeed.ts** - Candidates running for office (24 total)
- **userSeed.ts** - Voters and EC staff (14 total)
- **voteSeed.ts** - Votes cast (4 total)

### Utilities
- **index.ts** - Central export point for all seed data
- **seed.ts** - Script to insert all seed data into database

## Data Overview

### Tables

| Table | Records | Notes |
|-------|---------|-------|
| Admin | 2 | System administrators |
| Constituency | 6 | 3 open, 2 closed, 1 open |
| Party | 5 | Political parties |
| Candidate | 24 | 4 per constituency |
| User | 14 | 12 voters, 2 EC staff |
| Vote | 4 | Only in closed constituencies |

### Closed Constituencies
- Bangkok 3
- Chiang Mai 2

These constituencies have voting closed and results published, with test votes included.

## Usage

### Running Seeds

```bash
# Using ts-node
ts-node src/db/seeds/seed.ts

# Using npm script (if configured)
npm run seed:data
```

### Importing in Code

```typescript
import {
  adminSeedData,
  constituencySeedData,
  partySeedData,
  candidateSeedData,
  userSeedData,
  voteSeedData,
} from './seeds';

// Use in your code
for (const admin of adminSeedData) {
  // Process admin...
}
```

## Default Passwords

All passwords in seed data are hashed with bcrypt using the same cost factor (10).

For testing purposes, the plaintext passwords are:
- Admin accounts: `password123`
- Voter accounts: `voter123`
- EC staff accounts: `ec123`

## Data Relationships

```
Constituency (6)
├── Candidate (24) - 4 per constituency
│   └── Party (5)
└── User (14) - voters assigned to constituency
    └── Vote (4) - votes recorded in closed constituencies
```

## Adding More Seed Data

To add more records:

1. Edit the appropriate `.seed.ts` file
2. Add records to the exported array
3. Run `seed.ts` to update database

Example:
```typescript
// In userSeed.ts
export const userSeedData: UserSeedRecord[] = [
  // ... existing records
  {
    national_id: '1234567890020',
    password: '$2a$10$...',
    title: 'Mr.',
    first_name: 'New',
    last_name: 'User',
    address: 'New Address',
    role: 'VOTER',
    constituency_id: 1,
    created_at: new Date(),
  },
];
```

## Notes

- All IDs are auto-generated by the database sequences
- Passwords are hashed before being inserted into the database
- Dates are stored as UTC ISO strings
- Image URLs use placeholder service for testing
- Vote records are only created for closed constituencies
